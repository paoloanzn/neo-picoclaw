# App Structure - PicoClaw Custom Fork
# Last updated: 2026-03-17

## Repositories
- Upstream: https://github.com/sipeed/picoclaw (pinned at v0.2.3)
- Fork: https://github.com/paoloanzn/picoclaw

## Architecture
- Language: Go
- Upstream base: sipeed/picoclaw v0.2.3 (SHA 6f304362f6747221ad17f1437946f062e7b6e805)
- Fork strategy: Patch-based (git format-patch / git am --3way)
- Build: go build ./cmd/picoclaw/

## Patch System
- Config: UPSTREAM.conf (pinned upstream version)
- Registry: PATCHES.md (human + AI-readable intent/scope/risk)
- Patch files: patches/*.patch (applied in numeric order)
- Scripts: scripts/apply-patches.sh, generate-patches.sh, upgrade-upstream.sh, ai-regenerate-patch.sh
- CI scripts: scripts/ai-regenerate-patch-ci.mjs (Claude Agent SDK), scripts/package.json

## Active Patches (v0.2.3-custom.1)
- 001-add-sop-framework: SOP execution framework in workspace template + system prompt
- 002-relax-exec-guard: Relaxed exec tool safety guard for usable workspace restriction
- 003-ci-autofix-agent-sdk: Cloud-based AI patch auto-fix pipeline
- 004-fix-subagent-tools: Fix subagent empty tool registry (upstream PR #1711)

## CI Workflows
- .github/workflows/patch-ci.yml: Validates patches on push/PR
- .github/workflows/upstream-watch.yml: Daily cron, detects new upstream tags
- .github/workflows/patch-autofix.yml: AI auto-fix pipeline (Claude Agent SDK)

## Environment Variables Required
- ANTHROPIC_API_KEY (GitHub secret, for patch-autofix.yml CI workflow)
